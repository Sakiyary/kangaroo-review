# Mock Exam Page Ralph Context

## Task Statement

Add a new "模拟卷 / Mock Exam" page to the Kangaroo review site. The page should generate one English mock exam based on the complete review-class recording/outline as the highest-priority syllabus and the past-paper corpus as supporting evidence, with newer and adjacent-course papers weighted higher. It should model the paper format and wording after `raw/软统2022试卷.pdf`, ignoring design-pattern-only sections.

## Desired Outcome

- A new page entry in the site navigation for `#mock` or equivalent.
- An English-only mock paper with realistic question ordering, difficulty, probability/rationale, and point allocation.
- Chinese-only answer key with expandable answers, related topic expansion/jump affordances, and SVG reference diagrams for drawing/design questions.
- Export controls for the paper and the answer key as Markdown and printable PDF.
- The new page should support existing checklist/metrics/comment conventions where appropriate.
- The implementation should be locally verified, visually checked, committed, pushed, and deployed if complete.

## Known Facts / Evidence

- Existing app is static frontend plus lightweight metrics backend; primary UI code lives in `site/app.js`, content in `site/content.js`, question data in `site/data/questions.js` / JSON, styles in `site/styles.css`, and smoke tests in `tools/`.
- Existing pages are controlled by the `pages` set and `renderCurrentPage()` in `site/app.js`.
- Recent language-switch scroll anchoring added stable `data-scroll-key` and a regression script.
- Existing diagrams are SVG assets under `site/assets/diagrams`; current question rendering already supports diagram cards and drawing guides.
- Source hierarchy previously established: full review recording + review slides as the main syllabus, then recent current-course/adjacent-course past papers, then older papers and 2025 supplementary notes.

## Constraints

- Mock questions must be in English only.
- Answers must be in Chinese only.
- Do not i18n the mock paper/answer content.
- Ignore design-pattern-only content from the adjacent Software System Design paper.
- Use the complete review-class outline as the central scope; do not revive out-of-scope historical material as core exam content.
- Use SVG for answer diagrams and verify rendered diagram quality with multimodal/browser checks.
- Preserve prior site features and constraints: mobile readability, no horizontal overflow, checklist/metrics consistency, and current route behavior.

## Unknowns / Open Questions

- Exact number and marks of the 2026 exam are not fully known; infer a plausible format from review scope and past-paper style, and make the rationale visible.
- `raw/软统2022试卷.pdf` may be image-based or partially OCR-able; inspect pages visually to ground format.
- Export-to-PDF can be implemented via browser print if client-side binary generation would add unnecessary dependencies.
